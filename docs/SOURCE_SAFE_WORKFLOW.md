# Source-Safe Storage Workflow

This workflow is defensive guidance for reducing harm if a device is copied, inspected, or opened under pressure.

## Separation Principle

Avoid carrying all related material in one local entry. Separate:

- source identity;
- communication details;
- evidence files;
- notes;
- publication drafts;
- operational context;
- travel documents.

Storing all related materials together can increase harm if one local entry is disclosed.

## Neutral Labels

Use labels that do not reveal sensitive people, roles, locations, or operational meaning.

Safer examples:

- Local Notes
- Travel Set
- Interview Notes
- Image Set
- Temporary Holding
- Research Material

Avoid labels that identify a person, role, location, classification, or disclosure meaning.

## Metadata

Before storing files, review whether they may contain metadata such as author names, camera model, GPS coordinates, creator application, embedded thumbnails, local paths, or original filenames.

Phantasm includes a local metadata risk check and a limited best-effort metadata reduction path. Metadata removal is best-effort and may not remove every embedded identifier from every file format.

## Retention

The safest sensitive data is data not carried.

- Store only what is necessary.
- Remove stale local entries after the trip or task.
- Avoid carrying old contact lists.
- Avoid mixing unrelated work in one local entry.
- Review contents before checkpoints, border crossings, and other inspection events.
- Use external key material for high-risk deployments.

This document does not provide advice for surveillance evasion, covert communications, or unlawful concealment.
